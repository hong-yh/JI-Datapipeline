Welcome to the design project by UM-SJTU Joint Institutesponsored by Beijing DataPipeline Limited

Today, with the growing of the internet, we can easily get large amounts of data from e-commerce platforms, online sensors, social media. And we need to detect the abnormal data point, like finding cheating behavior for online shipping sites, some retailers may use the clicking farming and fake review to increase their rate unfairly, which would surely harm the usersâ€™ experience and should be detected and removed. Also, we can use data collected to build weather forecast system, from the abnormal in data likes temperatures, winds, we can find the unusual climate phenomena. However, The great number of data makes it impossible for humans to review all of them. so we are going to design the system that can automatically detect the Abnormal data for stream data.


                      Figure 1 : The Anomaly Detection

To achieve this, we are going to build a Machine learning model and train it based on interesting data sets. In the project, there are mainly three parts, the input data re-sampling and prepossessing, C-LSTM training models, and the front-end alerting systems.

Data Resampling

We choose the power usage data as a test example. This is a highly time-series-related data set, it has a significant time pattern. Also, it has a long enough data duration to build a model with enough information. Data is recorded at 1 hour sample period over 4 years, which maintain the feature of the dataset and reduce the train time.

Deep Learning Model

We finally choose C-LSTM, a combination of Convolutional Neural Network(CNN) and Long Short-Term Memory (LSTM), as our model. This model combines the strengths of both architectures and demonstrated to be capable of achieving remarkable performance in time serial data analysis. C-LSTM utilizes CNN to extract features from a sequence of data. These features are then fed into an LSTM to obtain the temporal representation.

                      Figure 4 : Model Structure

Front End Alerting

The front-end design is separated into two parts: the representational state transfer application programming interface (REST API) and the user interface.
The REST API acts as a connection between the database and the user interface. It reads anomaly data points from the local database and returns a JSON file to the corresponding API route.
The structure of the web page can be illustrated as a document object model (DOM) tree. The DOM tree for this project's website is shown in figure 5. DOM element $<$script$>$ is rendered by running several JavaScript files. These files visit the API routes to get the anomaly information they need to show on the web page and insert the values into the HTML templates they render. Once the web page is rendered. a JavaScript file will send a request to the APIs per second to check updated anomalies. New anomalies will be inserted into the HTML templates and be rendered in the same way.
By using JavaScript, the front-end of the web application can generate real-time graphs that show the actual streaming data, as well as the predicted value, which makes the trending more intuitive.

                      Figure 5 : DOM Structure

Train Results

To quantitatively compare the performance of C-LSTM against a fully connected layer and traditional LSTM, the mean square error(MSE) is recorded throughout the training process. Figure 6 and Figure 7 show the MSE variation as the training goes. We find that in both data sets, the C-LSTM model has the least MSE for given training time.
